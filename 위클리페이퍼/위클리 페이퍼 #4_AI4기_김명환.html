--- ## 단원 1: 딥러닝과 머신러닝의 포함관계 ### 1.1 기본 개념 이해하기 머신러닝(Machine Learning)과 딥러닝(Deep Learning)의 관계를 이해하기 위해서는 먼저 인공지능(AI)의 전체적인 구조를 알아야 합니다.
인공지능(AI)
머신러닝(Machine Learning)
딥러닝(Deep Learning)
### 1.2 각 개념의 정의

인공지능(Artificial Intelligence)

정의: 인간의 지능을 모방하여 문제를 해결하는 모든 기술

예시: 규칙 기반 시스템, Expert System, 머신러닝 등

머신러닝(Machine Learning)

정의: 명시적으로 프로그래밍하지 않고도 데이터로부터 학습하여 성능을 개선하는 알고리즘

특징:

  • 데이터에서 패턴을 자동으로 학습
  • 새로운 데이터에 대한 예측 가능
  • 전통적인 알고리즘: Linear Regression, Decision Tree, SVM, Random Forest 등

딥러닝(Deep Learning)

정의: 인공 신경망을 여러 층으로 깊게 쌓아 복잡한 패턴을 학습하는 머신러닝의 한 분야

특징:

  • 다층 신경망 구조 (Hidden Layer 3개 이상)
  • 비선형 변환을 통한 복잡한 패턴 학습
  • 이미지, 텍스트, 음성 등 비정형 데이터 처리에 특화
## 단원 2: 주요 차이점 분석
구분 머신러닝 딥러닝
데이터 요구량 상대적으로 적은 데이터 대량의 데이터 필요
특성 추출 수동 특성 엔지니어링 자동 특성 추출
계산 복잡도 상대적으로 낮음 매우 높음 (GPU 필요)
해석 가능성 높음 (투명한 모델) 낮음 (블랙박스)
성능 구조화된 데이터에서 우수 비정형 데이터에서 압도적
## 단원 3: 하이퍼파라미터 최적화 ### 3.1 주요 하이퍼파라미터

🎯 Learning Rate

역할: 가중치 업데이트 크기 결정

권장값: 0.001 ~ 0.01

optimizer = Adam(learning_rate=0.001)

📦 Batch Size

역할: 한 번에 처리할 데이터 수

권장값: 16, 32, 64, 128

model.fit(X, y, batch_size=32)

🔄 Epochs

역할: 전체 데이터를 학습하는 횟수

권장값: 50 ~ 200

model.fit(X, y, epochs=100)

🚪 Dropout Rate

역할: 과적합 방지

권장값: 0.2 ~ 0.5

Dropout(0.3)
### 3.2 메모리 관리 전략

💾 메모리 최적화 방법

1. 배치 사이즈 조정

# 메모리 부족 시 배치 사이즈 줄이기
batch_size = 16  # 32에서 16으로 감소
model.fit(X_train, y_train, batch_size=batch_size)

2. 모델 구조 간소화

# 뉴런 수 줄이기
model = Sequential([
    Flatten(),
    Dense(128, activation='relu'),  # 256에서 128로 감소
    Dense(10, activation='softmax')
])

3. 그래디언트 어큐뮬레이션

# 작은 배치를 여러 번 누적하여 큰 배치 효과
accumulate_grad_batches = 4
effective_batch_size = batch_size * accumulate_grad_batches
